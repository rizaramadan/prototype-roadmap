## State Machine

```
applied → screening (recruiter reviews)
screening → assessment (assessment delegated)
assessment → interview (assessment completed + passed threshold)
assessment → rejected (below threshold)
interview → offer
offer → hired
offer → rejected
any non-terminal → rejected (CandidateRejected command)
```

## Invariants

- `(personId, jobId)` pair is unique — a person can apply once per job
- `stage` is monotonic — it cannot move backward
- `fitScore` is immutable once written by ScoringService
- `assessmentId` may only be set by AssessmentDelegationService

## Commands

| Command               | Guard                                                  |
|-----------------------|--------------------------------------------------------|
| AdvanceCandidateStage | targetStage must be forward in the state machine       |
| RecordHiringDecision  | candidateId must be in `interview` or `offer` stage    |

## Events emitted

| Event              | Trigger               |
|--------------------|-----------------------|
| CandidateApplied   | (application intake)  |
| CandidateHired     | RecordHiringDecision hired=true   |
| CandidateRejected  | RecordHiringDecision hired=false  |
