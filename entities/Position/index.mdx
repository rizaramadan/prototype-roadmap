## Ontology model

```
Core Behaviour (atom)
  → Competency (molecule: named cluster of Core Behaviours)
    → Position (compound: required Core Behaviour profile + weights)
```

A Position is expressed directly in Core Behaviours with minimum score thresholds and weights. Competencies are a human-readable grouping layer but the matching algorithm works on raw Core Behaviour scores.

## Invariants

- `coreBehaviourRequirements` must contain at least one entry
- Each `coreBehaviourId` must reference an existing CoreBehaviour
- `minScore` must be in [0, 1]
- `weight` values across all requirements must sum to 1.0
- Position title is not a unique key — the same title can exist at different levels
