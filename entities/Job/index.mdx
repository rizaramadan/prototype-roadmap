## State Machine

```
draft → open (PostJob command)
open → paused (manual pause)
paused → open (resume)
open → filled (CandidateHired event closes the job)
open → closed (manual close without hire)
```

## Invariants

- `positionId` is immutable after creation
- `tenantId` is immutable after creation
- A Job can only have one active `CandidateHired` event — filling the job auto-closes it
- `assessmentConfigId` override must be a subset of the Position's required Core Behaviours
