## State Machine

```
pending → started (person begins first instrument)
started → completed (all responses submitted via SubmitResponses)
started → expired (expiresAt passed without completion)
pending → expired (expiresAt passed before person started)
```

**Invariants:**
- `SubmitResponses` is rejected if `status = completed` or `status = expired`
- `completedAt` is immutable once set
- `personId` and `tenantId` are immutable after creation
- A single person may have multiple assessments — each is a distinct aggregate instance

## Commands

| Command           | Guard                                          |
|-------------------|------------------------------------------------|
| AssignAssessment  | personId must have active ConsentGranted       |
| SubmitResponses   | status must be `started`                       |
