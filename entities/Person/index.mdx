## Portability principle

A Person's data (assessments, awards, IDP records) is owned by the individual, not by any employer. When a person moves between tenants, the `PersonCoreBehaviourProfileUpdated` event travels with them — the new employer's consent gate determines which data they can read.

## Invariants

- `hashedNationalId` is immutable after registration
- `consentState` transitions: `none → granted → revoked → granted` (cyclic — consent is revocable and re-grantable)
- `coreBehaviourProfile` may only be written by ScoringService via `PersonCoreBehaviourProfileUpdated`
- A Person aggregate must exist before any Assessment can be assigned

## Commands

| Command        | Guard                                         |
|----------------|-----------------------------------------------|
| RegisterPerson | hashedNationalId must not already exist        |
| UpdateProfile  | personId must exist                           |
| GrantConsent   | consentState must be `none` or `revoked`      |
| RevokeConsent  | consentState must be `granted`               |

## Events emitted

| Event                             | Trigger              |
|-----------------------------------|----------------------|
| PersonRegistered                  | RegisterPerson       |
| ProfileUpdated                    | UpdateProfile        |
| ConsentGranted                    | GrantConsent         |
| ConsentRevoked                    | RevokeConsent        |
