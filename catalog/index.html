Every domain speaks Core Behaviours. This enables clustering, matching, and cross-platform intelligence.

🔗

Identity linking: A hashed national ID is the cross-domain person key. Each platform collects it at intake. Federated — no single source of truth.

Consent model: Per-platform, person-granted. Managed via bakat.id. Assessment results earned at Client A are retained in bakat.id and travel with the person to Client B upon explicit authorization.

Multi-tenancy: peningkatan.id and recru.id run as <client>.* subdomains. Events are tenant-isolated; Talentlytica holds an internal platform-wide subscription for analytics.

Person Domain

bakat.id

Portable person profile and passport. Consent control plane for cross-platform data portability. The person's permanent career record — survives employers.

bakat.id · siswa · beasiswa · award

Assessment Domain

ases.id

Core Behaviour signal ingestion and storage. The measurement engine — all assessments funnel here. VictoriaMetrics analogy: ingest, store, emit.

ases.id · shared platform service

Position Service

posisi.recru.id

Position and role taxonomy expressed in Core Behaviours. The canonical reference for what any role demands — Talentlytica-native equivalent of O*NET / SkillsFuture. A sub-domain of recru.id.

posisi.recru.id · position master data

HRD Intelligence Domain

peningkatan.id

Multi-tenant HR intelligence platform. Reads ases.id signals and turns them into decisions: VisMap, career paths, succession plans, IDPs, Interventions. Grafana analogy.

<client>.peningkatan.id · multi-tenant

Recruitment Domain

recru.id

Multi-tenant ATS and job portal. Manages the full hiring funnel — job posting, candidate pipeline, assessment delegation to ases.id, handoff to peningkatan.id on hire.

<client>.recru.id · multi-tenant

Key Cross-Domain Flows

Recruitment → Assessment → Hire

1
recru.id Job posted, candidate applies
Candidate applies via recru.id or an external ATS (ats.ClientX.com). Application triggers assessment delegation.
2
ases.id AssessmentAssigned
Candidate is invited via email. They log in or register on ases.id and complete the assessment. Raw Core Behaviour signals are stored.
3
PersonCoreBehaviourProfileUpdated emitted
ases.id emits the event. recru.id sees the fit score; bakat.id retains the result in the person's portable profile.
4
recru.id CandidateHired → peningkatan.id TalentOnboarded
On hire, recru.id emits CandidateHired. The client's peningkatan.id listens and onboards the person as a talent.

Data Portability — Person Moves Between Employers

1
ases.id Assessment done at Client A
Learning Agility and other Core Behaviour scores are measured and emitted as AssessmentCompleted.
2
bakat.id AssessmentResultRetained
Result is stored in the person's portable profile in bakat.id, independent of Client A's tenant.
3
bakat.id ConsentGranted to Client B
Person joins Client B, goes to bakat.id, and authorizes clientB.peningkatan.id to read their retained data.
4
peningkatan.id Consumes historical profile
Client B's peningkatan.id receives ConsentGranted and queries the person's portable Core Behaviour history from bakat.id to pre-populate VisMap.

Intervention Cycle

1
peningkatan.id HR identifies gap via VisMap
HR manager views talent's Core Behaviour profile against position requirements in the VisMap dashboard.
2
CreateIntervention command issued
HR logs a real-world improvement action. An IDP is assigned. This is a manual human decision, not automated.
3
ases.id Re-assessment triggered
After the intervention period, a new assessment is assigned to measure progress on targeted Core Behaviours.
4
peningkatan.id VisMap updated
New signals flow back. VisMap reflects improvement. Cycle repeats as needed.
