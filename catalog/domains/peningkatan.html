📈 Overview

peningkatan.id is the intelligence layer on top of ases.id's raw signals. It doesn't measure — it interprets, visualizes, and enables HR decisions.

📊

Grafana analogy: ases.id is the time-series database storing raw Core Behaviour measurements. peningkatan.id is the dashboard layer — HR managers configure views (VisMap), set up alerts (Interventions), and build panels (career paths, succession plans, IDPs) on top of those measurements.

👤

Human-in-the-loop by design: Interventions are manual. The system surfaces gaps and recommendations, but an HR manager must take the action. peningkatan.id supports human judgment — it doesn't replace it.

🏢 Multi-tenancy Model

AspectBehaviour
Event namespaceAll events prefixed with the tenant identifier. clientA events are invisible to clientB.
Data isolationTalent data, intervention records, and career paths are fully isolated per tenant.
Talentlytica internal subscriptionTalentlytica holds a platform-wide subscription to all tenant event streams for analytics and benchmarking (aggregate/anonymized).
Person data portabilityWhen a talent leaves, their Core Behaviour history is retained in bakat.id — not in the tenant. The tenant holds the working copy; bakat.id holds the canonical portable copy.

⚙️ Services

VisMapService

Visualizes a talent's Core Behaviour profile against their current and target position requirements. The primary HR intelligence dashboard.

read-heavy

InterventionService

Manages manual HR improvement actions. HR logs real-world interventions (coaching, training, reassignment) and tracks outcomes via subsequent ases.id measurements.

publishes eventsaccepts commands

CareerPathService

Defines career progression paths using Core Behaviour delta data from posisi.recru.id. Shows what to develop to move from current to target position.

publishes events

IDPService

Individual Development Plan management. IDPs are generated from Core Behaviour gaps (current vs. target position) and linked to Interventions.

publishes events

SuccessionService

Identifies and tracks succession candidates for key positions. Uses Core Behaviour fit scores from ases.id to rank readiness.

publishes events

TalentOnboardingService

Handles the transition from candidate (recru.id) to talent (peningkatan.id). Consumes CandidateHired and bootstraps the talent's profile in the client tenant.

consumes events

🔄 Intervention Workflow

The core HR action cycle — observe → act → measure.

1
Observe — HR reviews VisMap
HR manager identifies a Core Behaviour gap between the talent's current profile and their position requirements.
2
CreateIntervention — HR logs the action
HR documents the improvement action: coaching, training, mentorship, etc. Specifies which Core Behaviours are targeted.
3
Act — Real-world execution (off-system)
The intervention happens in the real world. peningkatan.id records and tracks it — it doesn't orchestrate it.
4
TriggerReassessment → ases.id
After the intervention period, HR triggers a reassessment. ases.id emits new Core Behaviour scores.
5
PersonCoreBehaviourProfileUpdated consumed
New signals update the VisMap. HR compares before/after. IDP gaps are recalculated. Cycle may repeat.
