📊 Overview

ases.id is a shared platform service — the measurement engine for the entire ecosystem. Every assessment, from recruitment screening to employee development, flows through ases.id.

⚡

VictoriaMetrics analogy: ases.id ingests raw signals (assessment responses) and stores them as Core Behaviour measurements. It doesn't interpret or act on the data — that's peningkatan.id's job. ases.id ensures measurements are accurate, durable, and queryable.

🔗

Cross-domain reach: ases.id is the only domain that directly touches all others. recru.id sends candidates here for hiring assessments. peningkatan.id sends employees here for development assessments. bakat.id retains the results. External ATS systems bridge in via email invitation.

⚙️ Services

IngestionService

Receives, validates, and stores assessment responses. Maps raw responses to Core Behaviour dimensions using the posisi.recru.id ontology.

publishes events

InvitationService

Manages assessment invitations sent via email. Handles both direct invites (from peningkatan.id) and bridged invites (from external ATS systems).

accepts commands

ScoringService

Computes Core Behaviour scores from raw responses. Scores are normalized against position requirements sourced from posisi.recru.id.

publishes events

IntakeService

Handles person onboarding within ases.id. Offers "use previous data or re-intake" for returning persons. Links to bakat.id profile via hashed national ID.

consumes events

🔀 Intake Flow

When a person arrives at ases.id, they go through intake that links them to the ecosystem.

1
Person arrives via invitation link
Invitation was sent by recru.id (hiring), peningkatan.id (development), or external ATS.
2
Identity resolution
ases.id checks if this person has an existing bakat.id profile via hashed national ID. If yes: "use previous data or re-intake". If no: full intake.
3
Consent check
If previous data exists, person decides whether to grant consent to link this assessment to their bakat.id portable profile.
4
Assessment begins
AssessmentStarted emitted. On completion: AssessmentCompleted and PersonCoreBehaviourProfileUpdated emitted.
