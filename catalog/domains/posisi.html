Position Service

posisi.recru.id

The canonical taxonomy of positions, roles, and the Core Behaviours that define them. A sub-domain of recru.id.

URLposisi.recru.id Ownerrecru.id (sub-domain) Analogous toO*NET · SkillsFuture — Core Behaviour-native

🏷️ Overview

posisi.recru.id answers: "What does this position actually require?" It breaks every role into its constituent Core Behaviours, enabling precise matching, career path design, and targeted assessment.

🔭

The right side of Talentlytica's mission: If bakat.id describes who a person is in Core Behaviour terms, posisi.recru.id describes what a position demands. ases.id measures the gap between the two.

🏢

Sub-domain of recru.id: posisi.recru.id is owned by the recruitment bounded context. recru.id uses it internally for job requirements; ases.id and peningkatan.id consume its ontology cross-domain for scoring and career pathing.

🌐

External reference alignment: posisi.recru.id is Talentlytica's native equivalent of O*NET and SkillsFuture. Where those systems use skills and competency frameworks, posisi.recru.id uses the Core Behaviour ontology — more atomic, more matchable.

⚛️ Core Behaviour Ontology

The shared kernel the entire ecosystem speaks. posisi.recru.id is the authoritative owner of this ontology.

🧬

Atom → Molecule → Compound

A Core Behaviour is the smallest observable unit — e.g., "Seeks feedback proactively".

Core Behaviours cluster into Competencies — e.g., "Learning Agility" is a cluster of adaptation-related Core Behaviours.

Competencies compose into Position profiles — e.g., "Senior Product Manager" requires specific competency weights, derived from weighted Core Behaviour clusters.

This enables mix-and-match career pathing: two positions sharing 70% of Core Behaviours can be bridged by a targeted IDP.

⚙️ Services

CoreBehaviourService

Authoritative CRUD for Core Behaviour definitions. Changes here ripple to ases.id and peningkatan.id via events.

publishes events

PositionTaxonomyService

Manages position definitions and maps them to weighted Core Behaviour clusters. Used internally by recru.id for job requirements and cross-domain by peningkatan.id for career paths.

publishes events

CompetencyService

Manages competency definitions as named clusters of Core Behaviours. Bridges the ontology for human-readable use — HR sees competencies, the system works in Core Behaviours.

publishes events

ExternalMappingService

Maintains mappings between posisi positions and external frameworks (O*NET SOC codes, SkillsFuture). Read-only reference — not authoritative.

ACL · read-only
