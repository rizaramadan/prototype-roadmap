Emitted when a competency is successfully created. PositionTaxonomyService listens to keep position requirement profiles consistent with the latest competency definitions.

## Envelope

| Field         | Type   | Notes                                                    |
|---------------|--------|----------------------------------------------------------|
| traceId       | uuid   | Propagated from originating request                      |
| correlationId | uuid   | Groups related events across services and workflows      |
| causationId   | uuid?  | ID of the event or command that directly caused this one |

## Schema

| Field             | Type     | Notes                                              |
|-------------------|----------|----------------------------------------------------|
| competencyId      | uuid     | New competency's identifier                        |
| name              | string   | Human-readable competency name                     |
| coreBehaviourIds  | uuid[]   | Core Behaviours that form this competency          |
| definedAt         | datetime | ISO 8601                                           |
