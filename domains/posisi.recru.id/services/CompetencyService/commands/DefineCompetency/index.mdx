Competencies group Core Behaviours under a human-readable label (e.g., "Leadership" = [CB_assertiveness, CB_influence, CB_decisiveness]). Each competency must reference at least one Core Behaviour defined in CoreBehaviourService.

## Envelope

| Field         | Type   | Notes                                                    |
|---------------|--------|----------------------------------------------------------|
| traceId       | uuid   | Propagated from originating request                      |
| correlationId | uuid   | Groups related events across services and workflows      |
| causationId   | uuid?  | ID of the event or command that directly caused this one |

## Payload

| Field             | Type     | Notes                                              |
|-------------------|----------|----------------------------------------------------|
| competencyId      | uuid     | Caller-supplied idempotency key                    |
| name              | string   | Human-readable competency name                     |
| coreBehaviourIds  | uuid[]   | 1..N Core Behaviours that form this competency     |
| description       | string?  | Optional narrative description                     |

{
  "$schema": "http://json-schema.org/draft-07/schema#",

  "properties": {

    "coreBehaviourIds": { "type": "array", "items": { "type": "string", "format": "uuid" }, "minItems": 1 },

  }
}
